# Day 7 – SQL: Related Data & Joins

## Topics Covered

### 1. A First Look At Related Data
- Understood how real-world data is spread across multiple tables.
- Learned why storing all data in a single table leads to redundancy and inconsistency.

### 2. Data Normalization – First Steps
- Learned the concept of data normalization.
- Understood how normalization helps reduce data duplication and improve data integrity.

### 3. Splitting Data Into Tables
- Learned how to divide large tables into smaller, logical tables.
- Understood the importance of separating entities like users, orders, and products.

### 4. Forms Of Data Normalization
- Got introduced to different normal forms (1NF, 2NF, 3NF).
- Learned the basic rules each normal form follows.

### 5. Our First Section Example
- Worked through a practical example to apply normalization concepts.
- Observed how raw data is converted into normalized tables.

### 6. Creating Tables With Relations
- Learned how to create related tables using PRIMARY KEY and FOREIGN KEY.
- Understood how relationships are established between tables.

### 7. Inserting Related Data
- Practiced inserting data into tables with relationships.
- Learned the correct order of insertion to maintain referential integrity.



## Summary
Today’s Learned database normalization fundamentals, splitting data into related tables, applying normal forms, creating table relationships, and inserting related data to ensure consistency and integrity.
